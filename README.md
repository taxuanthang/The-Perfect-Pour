# The Perfect Pour
Lưu ý :
- KHÔNG push trực tiếp lên main nhé :V.
- Commit rõ ràng, ghi tóm tắt mình làm gì trong lần code này.
- Chắc chắn là phải ở trên nhánh mình mỗi khi code nhé.
- Khi nào hoàn thành chức năng (task) gì mà cần đẩy lên main thì thực hiện theo Bước sau:

    Bước 1: Commit đoạn code vừa xong lên nhánh của mình

    Bước 2: Merge nhánh main về nhánh mình và fix conflict (Không fix bừa bằng cách “chọn bên mình”, hãy đọc kỹ code cả 2 bên để giữ logic đúng.)(Nếu khó quá thì alo ae vào giúp nhé)

    Bước 3: Kiểm tra lại các chức năng mà mình vừa làm.

    Bước 4: 
        + GitHub Desktop sẽ hiện nút:
        + “Create Pull Request”
        + Bấm vào đó → nó sẽ mở GitHub Web và tạo form PR sẵn.
        + Kiểm tra lại:
            Base branch: main
            Compare branch: nhánh của bạn
        + Ghi mô tả ngắn → Bấm Create Pull Request.

    Bước 5: Review & Fix (nếu có)
        + Team  sẽ xem PR của bạn.
        + Nếu có góp ý, bạn chỉ cần:
        + Sửa code trên nhánh của mình.
        + Commit + Push lại.
        + GitHub sẽ tự cập nhật PR, không cần tạo lại.

    Bước 6: Merge PR
        Khi PR được duyệt:
        Người có quyền merge (hoặc bạn, nếu được phép) bấm “Merge pull request”.
    Bước 7: Hoàn thành
